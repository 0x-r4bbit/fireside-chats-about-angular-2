Classes

ES6 classes are syntactical sugar over objects and prototypes.

class Car {
  constructor(name) {
    this.name = name;
    this.milesDriven = 0;
  }

  drive(miles) {
    this.milesDriven = this.milesDriven + miles;
  }
}

let bmw = new Car('BMW');

Annotations + Decorators

Annotations

Annotations are a declarative way of attaching metadata to code.

@Annotation
class SomeClass {
 ...
}
  • Allows meta programming
  • What is AtScript?
  • Introduced annotations

AtScript/Traceur output

var SomeClass = (function() {
  function SomeClass() {}

  return SomeClass;
});

SomeClass.annotations = [new Annotation()];
  • What is Traceur?
  • Traceur transpiles AtScript annotations to this

So what are decorators then?

Decorators (ES7 proposal)

Decorators look pretty much the same but are more general and can be implemented by the consumer.

@Annotation
class SomeClass {
 ...
}
function Annotation() {
  return function (target) {
    target.annotations = [...]
  };
}
  • Decorators are an actual standard proposal for es7
  • Supported in TypeScript compiler
  • More general, flexible, developer is in charge

Attention!

(ES7) Decorators are not Decorator Directives!

Problems with interpolations

Why can't we just do this?

  • Interpolations don't work for attributes like `src` and `href`
  • Needs to be escaped

Boolean attributes

Boolean attributes write to the element's property.

<button disabled>...</button>
  • Some attributes are boolean attributes
  • They write to element's properties
  • Need to be escaped as well
  • There's a difference between attributes and properties

So what's the difference between attributes and properties?

Properties and Attributes




var input = document.querySelector('input');
input.value; // 'Carmen'
var img = document.querySelector('img');
img.src = // 'path/to/img.png'

input.value = 'Pascal';
input.getAttribute('value'); // 'Carmen'
  • Elements have attributes and properties
  • Attributes can be used to set an initial value (must be string!)
  • Properties are updated along an element's lifecycle
  • Some elements reflect property changes back to attributes (e.g. Img, but not input)

Web Components have the same APIs!

Property Binding

Angular 2 binds to element properties instead of attributes in order to work with any element.

Using [] syntax, we can bind to element properties.
A binding value is always an expression.

  • Angular 2 needs to be able to pass other types then strings to elements
  • Therefore, angular binds to properties rather than attributes
  • Properties can have any value
  • Prevents custom elements from accidentally reading literal expressions.
  • Works with any element (native, custom, polymer, ...)

My editor doesn't like it!

  • What if my editor and|or html preprocessor has problems with that syntax?

Canonical Syntax

Angular 2 supports a canoncial version of the property binding syntax to prevent possible problems with editors and server-side preprocessors.

bind-* is equivalent to [*]

  • Since [] is just a shorthand syntax, we can also write the canonical form
